# AI 彩票分析工具 — 自动更新实时数据 设计方案

> 版本：v1.0 | 日期：2026-07-04 | 状态：待实现

---

## 1. 目标

将当前纯离线 HTML 改造为 **GitHub Actions 定时更新 + GitHub Pages 在线部署** 架构，实现：

- 每期开奖后自动拉取最新数据
- 自动重跑三模型（随机森林 / 朴素贝叶斯 / 蒙特卡洛）
- 自动部署最新 HTML，用户通过 URL 直接访问

---

## 2. 整体架构

```
┌─────────────────────────────────────────────────┐
│                  GitHub 仓库                      │
│                                                   │
│  ┌──────────────┐    ┌──────────────────────────┐│
│  │   Actions     │    │      Pages (部署)        ││
│  │  (每天凌晨)   │    │                          ││
│  │               │    │  index.html              ││
│  │  crawl.py    ─┼───▶│  data/ssq_data.json      ││
│  │  model.py    ─┼───▶│  data/dlt_data.json      ││
│  │               │    │  data/ssq_model.json     ││
│  │  git commit   │    │  data/dlt_model.json     ││
│  │  git push     │    │                          ││
│  └──────────────┘    └──────────────────────────┘│
│                                                   │
│  用户浏览器 ── https://xxx.github.io/lottery-ai ──│
└─────────────────────────────────────────────────┘
```

两条核心流水线：

1. **数据更新流水线**（Actions）：每天凌晨 → 爬取最新期号 → 去重追加 → 重跑模型 → commit + push → 触发 Pages 部署
2. **前端渲染流水线**（浏览器）：用户访问 Pages URL → HTML `fetch` 同源 JSON → ECharts 渲染

## 4. Actions 工作流

### 4.1 触发条件

- **定时触发**：每天 UTC 18:00（北京时间凌晨 2:00），cron `0 18 * * *`
- **手动触发**：workflow_dispatch，支持调试时手动执行

### 4.2 执行步骤

| 步骤 | 内容 | 容错策略 |
|------|------|----------|
| 1. Checkout | 拉取仓库代码 | - |
| 2. Setup Python | Python 3.11 + 安装依赖（requests, scikit-learn, numpy） | pip 缓存加速 |
| 3. crawl.py | 分别爬取双色球（cwl.gov.cn）和大乐透（sporttery.cn），去重追加到 data/ 目录 | 任一源失败不影响另一个；单个源最多重试 3 次，间隔 5 秒 |
| 4. model.py | 基于更新后的数据重算三模型评分 | 仅 crawl 有新增数据时才执行 |
| 5. Diff check | `git diff --quiet data/` 判断是否有数据变更 | 无变更则跳过后续步骤，workflow 正常结束 |
| 6. Commit & Push | 提交 data/ 变更，推送至 main 分支 | commit message 包含日期和新增期号 |

### 4.3 关键参数

- 超时限制：10 分钟
- 时区：Asia/Shanghai
- Python 依赖缓存：`pip cache`

## 5. 前端改造

### 5.1 数据加载方式

| 改造前 | 改造后 |
|--------|--------|
| 内联 `<script>var SSQ_DATA = [...]</script>` | `fetch('./data/ssq_data.json')` |
| 数据嵌入 HTML 中（139KB） | 数据独立存放，HTML 体积大幅减小 |

### 5.2 加载流程

```
页面打开 → 显示 loading 骨架屏 → 并行 fetch 4 个 JSON
   ├─ 全部成功 → 隐藏骨架屏 → 渲染 UI → 用户交互
   └─ 任一失败 → 显示错误提示："数据加载失败，请刷新重试"
```

### 5.3 加载细节

- 首次加载显示骨架屏（灰色占位块动画），1 秒内可完成
- 4 个 JSON 并行 fetch，不加排队
- 错误重试 1 次，间隔 2 秒
- 现有 UI 全部保持不变：导航、图表、AI 模型面板、推荐号码、日期筛选、单号钻取、重新生成、导出 CSV

## 6. 部署与域名

### 6.1 GitHub Pages

- 仓库 Settings → Pages → Source: `Deploy from a branch` → `main` → `/ (root)`
- 默认域名：`https://<username>.github.io/lottery-ai/`
- 每次 Actions push 后自动部署，延迟约 30~60 秒

### 6.2 自定义域名（可选）

- 仓库根目录放置 `CNAME` 文件，内容为自定义域名（如 `lottery.example.com`）
- DNS 添加 CNAME 记录指向 `<username>.github.io`
- GitHub 自动签发 Let's Encrypt HTTPS 证书
